PIXEL PIPELINE · REFERENCE MANUAL

A three-stage pipeline: load or generate a source image → dither it to 48×48 → run it as a particle simulation. Each of the 2304 pixels carries five fundamental properties (r, g, b, x, y) that seed and drive every particle.

A · IMAGE — SOURCE STAGE

Define what the 48×48 pixel matrix looks like before dithering.

B · DITHER — QUANTISATION STAGE

Converts the source to a 48×48 binary/quantised image using error-diffusion or ordered dithering. This defines the particle colours.

algorithmdescription
FLOYD-STEINBERGClassic serpentine error diffusion. Smooth tonal gradients with fine grain. Best for photographic sources.
BAYER 44×4 ordered threshold matrix. Regular dot-screen pattern. Fast, deterministic.
BAYER 88×8 ordered matrix. Finer tonal steps than Bayer 4, smoother transitions.
ATKINSONApple Macintosh-era error diffusion. Propagates only 6/8 of the error, preserving highlights. Higher contrast than Floyd-Steinberg.
THRESHOLDHard cut at the threshold value. No diffusion. Maximum contrast, pixelated feel.
RANDOMStochastic threshold per pixel. Noisy, grainy film-like texture.
controldescription
thresholdMidpoint of the black/white decision. Lower = more white pixels. Default 0.50.
noise mixAdds random jitter to the threshold. Increases grain and breaks up regular patterns.
keep colorWhen ON, the dithered lightness pattern is applied to the original colour channels instead of producing greyscale. Particles retain source hue.
invertFlips all channel values after dithering.

Hover over the 48×48 dithered preview to inspect any pixel's exact (r, g, b, x, y) properties in the panel on the left. The colour band shows relative R, G, B weights.

C · SIMULATE — PARTICLE STAGE

Each of the 2304 dithered pixels becomes a particle seeded with its (r, g, b, x, y) and animated by a physics mode.

View modes

viewdescription
1 : 1Single 480×480 simulation with full controls panel. Cursor deformation active.
10 × 10 GRID100-cell grid: 10 simulation modes (columns) × 10 visual property variations (rows). All cells run at gentle speeds. Hue rotation separates rows visually. No post FX. Controls panel hidden.
FULLCanvas fills the window as a perfect square of pixels. Use the PX SIZE slider (2–32 px/cell) at the bottom to scale. Controls panel hidden in full mode.

Segmentation

Divides the 48×48 field into independent regions. Each segment runs its own physics centre, vortex axis, and phase offset.

moderegions
NONESingle field, all 2304 particles share one physics zone.
COLS 2/4/82, 4, or 8 vertical strips of 24, 12, or 6 columns each.
ROWS 2/42 or 4 horizontal bands.
GRID 2×2 / 4×44 or 16 equal square zones.

Simulation modes

modedescription
FLOWfBm noise velocity field drifts continuously. Particles follow organic, curling paths. Return force pulls toward home position.
VORTEXOrbital spin around each segment's centre. Inward pull creates a spiral. Noise jitter disrupts perfect circles.
WAVETravelling sine wave displaces particles from home. Phase offset cascades across the field.
SCATTERParticles repel from their home position. Bounce off segment walls. Noise drives random walk.
FORMATIONStrong home attraction with noise jitter. Particles oscillate around original pixel positions. Shows image structure clearly.
CASCADEGravity rain: particles fall, wrap to top on exit. Noise causes lateral drift.
ORBITEach particle orbits at a radius proportional to its original distance from segment centre.
BREATHERadial pulse in/out from segment centre. All particles expand and contract in a single rhythm.

PARASITE · HOST WINDOW

A second image is loaded as a "host." The live simulation output feeds into a cellular automaton that infests the host image pixel by pixel in real time.

How infestation works

  • Each tick, the sim canvas is sampled at 48×48 native resolution.
  • A pixel luminance value is computed per cell. If it exceeds the activation threshold, that cell is "active" in the CA.
  • Active cells that are already infested attempt to spread to their 4 Von Neumann neighbours with probability spread rate × luminance.
  • Each spread event copies the sim's colour with colour mutation random drift — the infection colour evolves as it propagates.
  • The final output blends host pixel colour with infested sim colour, weighted by infestation level × blend host/sim.
  • At full infestation (>90%), pixels receive an additional colour boost from their infesting source.
controldescription
spread rateCA propagation probability per tick (0.001–0.5). The key speed parameter.
activation threshMinimum sim brightness to trigger spreading. High = only bright sim pixels infect.
blend host/simHow strongly the sim colour replaces host colour at full infestation. 0 = host invisible, 1 = full replacement.
infestation decayHealing rate per tick. 0 = infection grows forever. >0 = cells slowly recover.
colour mutationPer-spread random drift on the infecting colour. High = colour evolves far from sim source.
seed densityFraction of cells seeded at random on reset. Adjusting re-seeds immediately.
invert activationWhen ON, dark sim pixels infect instead of bright ones.
sync to sim speedRuns CA every other tick instead of every tick. Keeps CA coupled to sim rate.
RESETClears the infestation map and re-seeds random starting points at current seed density.

The parasite window is only visible in 1:1 view. It is hidden in 10×10 GRID and FULL modes.

TEMPLATE SAVE / LOAD

All simulation settings can be saved to a .txt file (human-readable JSON) and reloaded.

  • Type a name in the template name field, then click SAVE SETTINGS .TXT to download.
  • Click LOAD SETTINGS .TXT to open a saved file. All sliders, buttons, and checkboxes sync instantly.
  • The template file stores: dither algorithm + parameters, simulation mode + physics values, rendering mode + visual parameters, post FX settings, cursor deform settings, and full-mode pixel size.
  • Templates do not store the image data — only the settings. Reload your source image separately.

TECHNICAL NOTES

  • The canvas is always 48×48 internal pixels. Display scale is CSS-only — the simulation always runs at native 48×48 resolution.
  • Value noise uses a 256×256 permutation table. fBm (fractal Brownian motion) stacks 3 octaves.
  • All 2304 particles update every frame. Performance depends on simulation mode and browser.
  • The 10×10 grid maintains 10 independent particle systems (one per column) and renders 100 display canvases per frame — expect lower frame rates on older hardware.
  • Template files are plain JSON with a _meta block containing name, version, and timestamp.
