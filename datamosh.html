DATAMOSH // BIO-GROWTH ENGINE
WHAT THIS IS
Real datamoshing: frame bleeding. Each frame is not drawn fresh — it samples pixels from the previous frame at displaced positions, creating smear and bleed artifacts. The displacement field is driven by a Gray-Scott reaction-diffusion system, a biological growth simulation that produces coral, spot, maze, and filament patterns.
GRAY-SCOTT REACTION-DIFFUSION
Two chemicals U and V interact: U feeds V, V autocatalyses. The equations are du/dt = Du·∇²u − uv² + F(1−u) and dv/dt = Dv·∇²v + uv² − (F+k)v. F (feed rate) and k (kill rate) determine the pattern type. The gradient of V at each pixel becomes the displacement vector for frame bleeding.
PRESETS
coral F=0.0545 k=0.062 — branching structures, spreading tendrils
spots F=0.035 k=0.065 — mitosis-like cell division, dot patterns
maze F=0.029 k=0.057 — labyrinthine channels, slow crawl
worms F=0.046 k=0.063 — writhing filaments, high motion
bubbles F=0.098 k=0.057 — foam decay, rapid diffuse spread
FRAME BLEED (DATAMOSH)
Bleed = how much of the previous frame persists (0.80 = heavy mosh, 0.97 = subtle). Displacement = how far pixels are thrown by the RD gradient per frame. High bleed + high displacement = aggressive pixel smear. The infection front (high V) causes maximum bleed; settled areas let the host image through.
30-SECOND LOOP
The animation runs for 30 seconds then rebirths: the RD system reseeds from the same initial conditions, and the ghost buffer cross-fades back to the host image over 1.5 seconds. The loop count increments. Each loop is identical given the same seed.
THE NOISE KEY — AND WHY IT IS THE CURE
When you click Capture Key, the current RD field (V-channel) is rendered as a 64×64 fluorescence image — this is the noise evolution snapshot. The parameters (F, k, seed, displacement, bleed, generation count) are steganographically embedded in the LSBs of its pixels.

Why it is the cure: Given the key, you can reconstruct exactly the same RD field at exactly the same generation. That field gave you the displacement vectors that distorted the host. Apply those same vectors in reverse (negate dx/dy), scan the infected image back through the anti-displacement, and the host is recovered. The RD field is deterministic — same seed + same parameters + same generation count = identical displacement field = perfect inverse.

WORKFLOW
1. Load a host image (optional — runs on black if none)
2. Choose a preset or dial in F/k manually
3. Click Play — watch the infection grow for up to 30s
4. Click Capture Key at any point — freezes the key at that generation
5. Save infected frame (download canvas) + save key (PNG or .txt)
6. To cure: load infected image as host, load key, click Apply Cure
NOTES
All computation runs client-side. Key PNG must be saved losslessly (PNG, not JPEG). The RD simulation runs at 128×128 with 6 steps per animation frame at ~30fps.
