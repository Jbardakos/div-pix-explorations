━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. PREPARE INPUT IMAGE
   • Resize to 256×256 px (1:1 aspect). Nearest-neighbor for pixel art,
     Lanczos for photo sources. Desaturate if using as luminance map.
   • Export as PNG (lossless).

2. LOAD AS FACE DRIVER
   • The rawB array (16×16 brightness blocks) will be derived from
     your image's average luminance per cell.
   • Bright areas → high noise value → dense patterns (filled blocks ██)
   • Dark areas → low noise value → sparse patterns (dots, empty)

3. APPLY TO LAYERS
   • Enable ◈ FACE DRIVER on any layer to use image brightness
     instead of Perlin noise as the pattern-selection source.
   • Layer 10 defaults to face-driven. Enable on layers 1–4 for
     coarse-to-fine image structure preservation.

4. BLOCK SIZE STRATEGY
   • L1 bs=64 → mega-blocks, raw form impression
   • L2 bs=32 → coarse structure
   • L3-L4 bs=16 → medium detail
   • L5-L7 bs=8 → fine texture
   • L8-L10 bs=4 → micro-detail / grain

5. BLEND MODE STRATEGY
   • Base layers: ADD — accumulates brightness
   • Mid layers: SCR (screen) — keeps highlights, drops blacks
   • Accent layers: XOR — creates edge-detected interference
   • Final face layer: OVL (overlay) — preserves image contrast

6. NOISE MODE
   • DIRECTIONAL: noise scrolls in x/y — good for flowing fields
   • EVOLUTIVE: noise coordinates rotate in a circle — no net drift,
     patterns mutate in place. Ideal for portrait-based compositions.

7. GLITCH SETTINGS
   • TEAR on base layers adds horizontal scan-line disruption
   • STRIPE adds vertical column drift
   • INVERT creates probability inversion zones
   • Use GRIT (0–100) to control inversion threshold

8. EXPORT YOUR WORK
   • GRID mode shows 4 simultaneous output frames (different time states)
   • 4× mode (1024px) for screen-quality output
   • Screenshot any frame for final image capture
